# Endpoint Schemas

Contract-level request/response references for high-traffic Aionis endpoints.

This page should be used after you already know which product line you are touching. It is not meant to replace the group pages in [Memory APIs](/api-reference/memory), [Policy APIs](/api-reference/policy), or [Replay APIs](/api-reference/replay). Its job is narrower: help you jump directly to the exact request and response shape once your integration question is already specific.

In practice, most teams should read endpoint pages in small clusters. If you are proving the basic memory wedge, start with `write`, `recall_text`, and `context/assemble`. If you are trying to govern tool choice, start with `rules/evaluate` and `tools/select`. If you are investigating reuse, failure reconstruction, or controlled re-execution, then move into the replay endpoints.

That matters because endpoint pages are strongest when they answer one exact question at a time. They are not the best way to learn the whole product from scratch.

## Fast cluster examples

### Memory cluster

```bash
curl -sS "$BASE_URL/v1/memory/write" -H 'content-type: application/json' -d '{"tenant_id":"default","scope":"docs_index","nodes":[{"type":"event","memory_lane":"shared","text_summary":"hello"}]}' | jq
curl -sS "$BASE_URL/v1/memory/recall_text" -H 'content-type: application/json' -d '{"tenant_id":"default","scope":"docs_index","query_text":"hello","limit":5}' | jq
```

### Policy cluster

```bash
curl -sS "$BASE_URL/v1/memory/rules/evaluate" -H 'content-type: application/json' -d '{"tenant_id":"default","scope":"docs_index","run_id":"run_001","context":{"intent":"support_triage"}}' | jq
curl -sS "$BASE_URL/v1/memory/tools/select" -H 'content-type: application/json' -d '{"tenant_id":"default","scope":"docs_index","run_id":"run_001","candidates":["router","email_sender"]}' | jq
```

### Handoff cluster

```bash
curl -sS "$BASE_URL/v1/handoff/store" -H 'content-type: application/json' -d '{"tenant_id":"default","scope":"docs_index","anchor":"task_001","handoff_kind":"task_handoff","summary":"continue","handoff_text":"next step"}' | jq
curl -sS "$BASE_URL/v1/handoff/recover" -H 'content-type: application/json' -d '{"tenant_id":"default","scope":"docs_index","anchor":"task_001","handoff_kind":"task_handoff"}' | jq
```

If you are still deciding which cluster to read first, use the simpler rule. Continuity and recall problems point to memory, uncontrolled tool behavior points to policy, cross-session or cross-agent continuation problems point to handoff, and expensive rediscovery or rerun work points to replay.
