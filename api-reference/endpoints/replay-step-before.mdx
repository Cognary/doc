# POST /v1/memory/replay/step/before

Use this endpoint when a host runtime records replay state before a step executes.

## What it is for

1. low-level replay instrumentation
2. step-by-step capture before execution
3. richer replay reconstruction later

## Minimal request

```bash
curl -sS "$BASE_URL/v1/memory/replay/step/before" \
  -H 'content-type: application/json' \
  -d '{
    "tenant_id":"default",
    "scope":"docs_replay",
    "run_id":"replay_run_001",
    "step_id":"step_001",
    "tool_name":"ticket_router"
  }' | jq
```

## Example response

```json
{
  "ok": true,
  "run_id": "replay_run_001",
  "step_id": "step_001",
  "phase": "before"
}
```

## Choose this route when

Use `step/before` only in low-level replay instrumentation flows. It is not a general replay entry point.
